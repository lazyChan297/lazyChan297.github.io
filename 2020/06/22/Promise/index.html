Promise

Promise是一个处理异步函数非常好的类,可以通过new Promise(handle)构造它的实例。
只有当你等待的函数有结果后，它还会进一步处理。
Promise一共有三种状态pending（等待中）,即没有回调结果时；resolve(成功)；reject(失败);
它的状态变化只能是 pending –> resolve 或者 pending –> reject;
一旦它的状态改变,就不可能回到原来的状态。顾名思义Promise

Promise的用法

接收一个handle函数作为传入参数,handle里的代码会立即执行。通常是在handle中存放判断状态成功或失败的代码。
handle接收resolve和reject两个函数参数，分别处理成功与失败的回调
resolve和reject则是会等候一段时间后才会执行,例如说网络请求有返回后、倒计时结束后等等

Promise.then()

1、Promise.then()接受的两个参数就是handle的resolve和reject(非必传)函数,如果传入的不是函数,会发生值穿透
2、返回值仍然是一个Promise的实例，所以支持链式调用，只有等上一个then状态改变后，才会执行下一个，
不管经过多少次调用,最后获得的返回值还是resolve传入的值或者上一个then的返回值，如果自定义返回了别的类型，也会将它封装成一个Promise实例

Promise.catch()

1、当promise的状态返回失败, Promise.then()没有传入reject时,catch()会捕获到异常

2、resolve或reject出现的错误,catch()会捕获到异常

为了防止promise状态失败但是没有传入reject回调,可以定义window.addEventListener(‘unhandledrejection’, event => { }) 监听事件进行相应处理

Promise.all()

当需要等待多个并行的异步请求都完成再回调时，可以使用.all([])
只有当所有的处理都resolve.all才会被resolve，返回一个包含所有请求的值的数组
只要有一个请求被reject，那么就会返回reject的值
多个请求之间不会相互阻塞，可以同时进行

Promise.race()

race()与all()相同点在于同样接受一个promise请求数组作为参数，都会并行执行
不同点在于只要哪一个promise率先完成，就会返回第一个完成的值
